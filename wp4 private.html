My Experience with Adobe Dreamweaver

 Dreamweaver is to fabricate and forge your website before your publish it, unlike Fireworks which is based on designing and planning. Coding is a major tool in dreamweaver that distinguishes it from fireworks. In terms of editing the page you could easily execute alternations since you could adjust it manually by coding. A cool tweak I am really fond and glad about dreamweaver is that is has this split screen function between coding and design; you can edit a code that will automatically execute the alterations onto the actual design and it makes it very efficient to reach to what you want. Another convenient tool is the laser linking, which really helped me link textual and graphical navigations- rather than using coding such as <a href…/>.  Since this is a HTML based, there is no exporting procedure, you can edit and directly preview a live version. More about exporting, dreamweaver has an excellent feature to connect your website to a server, for this project it would be “theeaglepages.” This made it much easier to publish things directly.

Advantages

  • HTML file without export
  • Automatically generate the code of manually placed objects through the design window.
  • Option to select- either CSS or HTML
  • Finder at the right side for designated project folder
  • Dreamweaver templates function. E.g- .dwt files that can be applied to all pages.
  • Easier for CSS styling
  • Allows users to drag images directly from the list of files in the folder on the side
  • Fireworks doesn't work well for printing your designs. Since the dots.per.inch is relatively lower. (Even though we learnt to change it, we dont really do it), the images come out a little bitmapped.
  • Can change CSS altogether as a class/tag/div using RULE: all titles had the same properties by adding the same rule to each AP div
  • Semi-automatically writes codes for you
  •  Split screen mode between code and design

Disadvantages

  • Many functions and options to opt for which can make it slightly confusing.
  • Sometimes manually changing your code does not work.
  • Interpreting codes can get a tad bit tedious.

My Experience with Adobe Fireworks

Fireworks is a straightforward and user friendly program application in which most of the work can be done by your mouse- drag & drop. It a simple when designing wireframes and storyboards because Fireworks is more of a graphical app. What I mean by graphical app is that it does not require HTML nor CSS to design, which makes it great to make blueprints or simple models to present before starting the actual project. It is my first year learning CSS and HTML coding, so fireworks did make things easier because it had features such as enabling and hiding layers that I am familiar from computer 1C. What I have regard for fireworks is after have finalised your blueprint, to export and create the html file you have to slice up your content. The procedure is rather simple, nevertheless If you decide not to slice up the images and your categorised content then when you view it on a  preview it on a browsing platform- it is just one big picture. The big picture disallows you to make slight changes you might want to make onto your work. From my experience, Fireworks is excellent for working with graphical related things on your website and to polish last minute changes.

Advantages

  • No CSS nor HTML coding.
  • Simple mouse clicking functions
  • Uncomplicated image optimisation.
  • Easily export into HTML format.
  • Accurate preview to understand its outlook and easily turn into a blueprint.
  • Easier for image related work. Such as size, format, pixel, blur, etc
  • Different layers can be hidden or shown- then locked for being able to edit easily
  • Different parts of your work can be saved/published seperately. For example, you can make buttons, but then just save the images for the buttons, boxes, or from a seperate layer.
  • Can crop/resize/edit images on the same software in the photo's respective file rather than going back to Fireworks if this process needs to be done when using Dreamweaver
  • Better for coordinating the layout of the whole page (spacing/alignment)

Disadvantages

  • Difficult to be precise with tiny details such as fonts because it is a picture
  • A lot of bugs in Fireworks, sometimes will crash without reason
